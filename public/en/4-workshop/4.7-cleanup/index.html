Cleanup Resources

Cleanup Resources

After completing the workshop, you should delete the resources to avoid unexpected costs.

Make sure you have backed up important data before deleting!

Deletion order is important: Resources must be deleted in the correct order to avoid dependency errors.

1. Delete Application Load Balancer

  1. Access EC2 Console → Load Balancers

  1. Select carbuyer-alb

  1. Click Actions → Delete load balancer

  1. Confirm by typing: confirm

  1. Click Delete

ALB must be deleted before Target Groups and EC2 instances.

2. Delete Target Groups

  1. In EC2 Console, select Target Groups

  1. Delete each target group: carbuyer-customer-tg, carbuyer-admin-employee-tg

  1. Select target group → Actions → Delete

  1. Confirm

3. Delete EC2 Instances

  1. In EC2 Console, select Instances

  1. Select both instances: carbuyer-customer-service, carbuyer-admin-service

  1. Click Instance state → Terminate instance

  1. Confirm deletion

Wait for instances to transition to Terminated state before continuing.

4. Delete NAT Gateway

  1. Access VPC Console → NAT Gateways

  1. Select carbuyer-nat-gateway

  1. Click Actions → Delete NAT gateway

  1. Confirm by typing: delete

  1. Wait 2-3 minutes for NAT Gateway to be completely deleted

5. Delete Elastic IP

  1. In VPC Console, select Elastic IPs

  1. Select Elastic IP created for NAT Gateway

  1. Click Actions → Release Elastic IP address

  1. Confirm

Unattached Elastic IPs will be charged. Delete immediately after removing NAT Gateway!

6. Delete API Gateway

  1. Access API Gateway Console

  1. Select API carbuyer-email-api

  2. Click Delete

  1. Confirm by typing: confirm

  1. Click Delete

7. Delete Lambda Function

  1. Access Lambda Console

  1. Select function carbuyer-email-service

  1. Click Actions → Delete

  1. Confirm by typing: confirm

  1. Click Delete

8. Delete CloudFront Distribution

  1. Access CloudFront Console

  1. Select distribution fcj-cloudfront-carbuyer

  1. Click Disable

  1. Wait a few minutes for status to change to Disabled

  1. Select distribution, click Delete

  1. Confirm deletion

CloudFront distribution must be Disabled before it can be deleted.

9. Delete S3 Bucket

  1. Access S3 Console

  1. Select bucket carbuyer-fcj-bucket

  2. Click Empty to delete all objects

  1. Confirm by typing: permanently delete

  1. Click Empty

  1. Go back, select bucket and click Delete

  1. Confirm by typing bucket name: carbuyer-fcj-bucket

  1. Click Delete bucket

10. Delete DynamoDB Tables

  1. Access DynamoDB Console

  1. Select Tables from left menu

  1. Select table to delete: DanhGia, DatLichKH, KieuDang, LoaiPhuKien, MauXe, NguyenLieuXe, PhuKien, Slider, ThuongHieu, TinTuc, User, XeOto

  2. Select table, click Delete

  1. Confirm by typing: confirm

  1. Verify result after deletion

11. Delete Security Groups

  1. Access EC2 Console → Security Groups

  1. Delete security groups: carbuyer-customer-sg, carbuyer-admin-sg

  1. Select security group → Actions → Delete security groups

  1. Confirm

  1. Repeat the process to delete carbuyer-alb-sg

12. Delete VPC Endpoints

  1. Access VPC Console → Endpoints

  1. Select endpoints to delete

  2. Select endpoint → Actions → Delete VPC endpoints

  1. Confirm

13. Delete VPC

  1. In VPC Console, select Your VPCs

  1. Select carbuyer-fcj-vpc

  2. Click Actions → Delete VPC

  1. Confirm by typing: delete

  1. Click Delete

Deleting VPC will automatically delete related resources: subnets, route tables, internet gateway.

14. Delete IAM Roles

  1. Access IAM Console → Roles

  1. Select roles: carbuyer-ec2-role, carbuyer-lambda-email-role

  2. Select Delete

  1. Confirm by typing: delete

  1. Click Delete

15. Delete IAM User

  1. In IAM Console, select Users

  1. Select user carbuyer-app-user
  2. Click Delete

  1. Select Deactivate access key

  1. Confirm and select Delete user

16. Delete Key Pair

  1. Access EC2 Console → Key Pairs

  1. Select carbuyer-keypair

  2. Click Actions → Delete

  1. Confirm
